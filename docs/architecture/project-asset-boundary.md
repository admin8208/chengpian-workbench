# 项目素材边界说明

## 目标

系统中的素材分为两类：

1. 公共素材库
2. 项目私有素材

公共素材库只作为数据源池使用，项目可以从中搜索和选择素材，但项目一旦真正使用素材，文件必须先进入项目目录，再绑定到项目或镜头。

## 设计原则

1. 公共素材库不应被单个项目流程污染
2. 项目绑定的资产必须是项目私有资产
3. 删除项目时，项目私有素材应随项目一起清理
4. 删除项目时，不应影响公共素材库原件

## 当前行为

### 公共素材库

以下入口仍然写入公共素材库：

1. 素材库页本地上传
2. 素材库页网盘导入
3. 明确调用 `/api/library/import` 的公共素材导入

这些资产具备以下特征：

1. `tag = library`
2. `project_id = null`
3. 文件路径位于 `data/assets/library/...`

### 项目私有素材

以下项目型流程现在会写入项目私有目录：

1. 项目页在线搜索后导入并绑定素材
2. 自动补素材流程
3. remix 项目型视频导入
4. 从公共素材库选择素材并绑定到项目时的复制副本

这些资产具备以下特征：

1. `project_id = 当前项目`
2. `tag = project_source` 或 `project_asset`
3. 文件路径位于 `data/projects/project_{id}/...`

## 关键接口

### 公共素材库接口

1. `POST /api/library/import`
2. `POST /api/library/assets`
3. `POST /api/cloud/import`

这些接口只用于公共素材库。

### 项目私有导入接口

1. `POST /api/projects/{project_id}/import-web`

该接口用于项目在线素材导入，文件直接写入项目目录。

## 绑定规则

项目在绑定素材时，不直接把公共素材库原件作为最终项目资产使用。

如果绑定的是非当前项目资产，后端会先复制到项目目录，再生成项目私有资产并绑定到 scene。

## 删除规则

删除项目时：

1. 删除项目记录
2. 删除项目私有资产记录
3. 删除 `data/projects/project_{id}/...` 下的项目文件
4. 保留公共素材库原件

## 历史数据说明

历史上项目在线导入曾错误写入公共素材库，造成素材库污染。当前主链已修正，但历史数据仍可能包含：

1. 被误沉淀到 `library` 的项目素材
2. 数据库记录仍在，但物理文件已不存在的失效公共素材记录

失效公共素材记录已经按当前环境做过清理。

## 后续维护建议

1. 新增项目型导入入口时，优先写项目私有目录
2. 只有明确的“加入公共素材库”动作才允许写 `library`
3. 如果后续要扩展项目从公共素材库选材功能，继续沿用“复制到项目后再绑定”的模型
